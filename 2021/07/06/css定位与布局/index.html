1. 定位

1.1 偏移属性

偏移属性用于描述定位元素各边相对于其包含块对应边的偏移量。

偏移属性 说明
top 顶部偏移量，元素上边线相对于父元素上边线的距离
right 右侧，子右边线相对于父右边线的距离
bottom 底侧，子下边线相对于父下边线的距离
left 左侧，子左边线相对于父左边线的距离

1.2 定位

1.2.1 静态定位 static

静态定位是元素在HTML文档流中的默认定位，无法通过偏移属性来改变元素位置。

元素的默认定位就是 {position: static}，故不用设置。

1.2.2 相对定位 relative

设置相对定位的元素是**相对于元素本身在标准文档流中的位置**，通过设置偏移属性来将其相对于本来位置的定位。

1.2.3 绝对定位 absolute

绝对定位是相对于父级元素的定位，会**脱离文档流且不占用空间，也成为脱标**。

设置绝对定位的元素的位置**相对于最近的已定位（设置position：absolute | fixed | relative）父级元素**、若其父级元素均没有定位，则相对于根元素<html>。

固定定位是**相对于浏览器窗口的绝对定位**，通过设置偏移属性来设置相对于浏览器的指定坐标。

不管浏览器滚动条如何滚动，窗口如何变化，该元素将会始终显示在浏览器窗口的固定位置。（完全脱标、不占有位置、只认浏览器）

1.2.5 粘性定位 sticky

这是一种结合了相对定位和固定定位的一种特殊定位，适用于一些特殊场景。

  • 当达到由偏移属性指定的条件时，将表现为固定定位fixed；

  • 当未达到条件时，将表现为相对定位relative

以上代码实现：当div[class=”sticker”]块下边框距浏览器窗口下边框超过100px时，表现为position: relative；当距离==100px时，表现为position: fixed

但该属性的兼容性不是很好，使用时需要特别注意。

其生效有一定限制，条件如下：

  1. 须指定 top, right, bottom 或 left 四个阈值其中之一，才可使粘性定位生效。否则其行为与相对定位相同。
    • 并且 top 和 bottom 同时设置时，top 生效的优先级高，left 和 right 同时设置时，left 的优先级高。
  2. 设定为 position:sticky 元素的任意父节点的 overflow 属性必须是 visible，否则 position:sticky 不会生效。
    • 如果 position:sticky 元素的任意父节点定位设置为 overflow:hidden，则父容器无法进行滚动，所以 position:sticky 元素也不会有滚动然后固定的情况。
    • 如果 position:sticky 元素的任意父节点定位设置为 position:relative | absolute | fixed，则元素相对父元素进行定位，而不会相对 viewprot 定位。
  3. 达到设定的阀值才表现为fixed。也就是设定了 position:sticky 的元素表现为 relative 还是 fixed 是根据元素是否达到设定了的阈值决定的。

1.3 定位的一些小细节

1.3.1 子绝父相

指：当子级希望依据父级来定位的话，子级是绝对定位，父级要用相对定位。

我们已经知道，绝对定位是将元素依据最近的已定位的父级元素进行定位。即子级是绝对定位，父级只要定位即可（无论是绝对定位、想对定位或是固定定位）；同时子级是绝对定位，会脱标，不会占有位置，故而可以放在父盒子的任意地方。而父盒子布局时需要占有位置，因而只能是相对定位——这就是子绝父相。
